# Join With Link Has No Link Entry and Take Seat No-Ops

> *This was generated by AI during triage.*

Status: ready-for-agent

Category: bug

## What is broken

The lobby CTA says `Join with a Link`, but it does not provide a way to paste or open an invite link. It navigates to an invite preview for the currently selected room instead. For the default room, `Take a Seat` has no visible effect.

## Reproduction

1. Enter the app as a guest.
2. From the lobby, click `Join with a Link`.
3. Observe the invite screen.
4. Click `Take a Seat`.

## Actual behavior

The screen shows `Gilt Room` as if the current player invited themselves:

`codex_tester invited you to the table`

There is no input for an invite URL or code. Clicking `Take a Seat` stays on the same screen with no error or progress.

## Expected behavior

`Join with a Link` should either:

- open a real join form where a player can paste an invite URL/code, or
- be removed/renamed until route-based invite links are implemented.

If joining fails, the app should show a specific error instead of silently doing nothing.

## Acceptance criteria

- [ ] The lobby join CTA has a real invite-code/link entry path.
- [ ] The invite join action cannot silently no-op when the selected room has no invite code.
- [ ] Failed invite joins show a specific user-facing error.
- [ ] Tests cover joining with a valid invite code and attempting to join without one.
