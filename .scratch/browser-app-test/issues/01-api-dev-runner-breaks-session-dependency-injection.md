# API Dev Runner Breaks Session Dependency Injection

> *This was generated by AI during triage.*

Status: ready-for-agent

Category: bug

## What is broken

Running the API with `pnpm --filter @cryptopoker/api dev` starts Nest successfully, but session-backed controller actions fail at runtime because constructor-injected dependencies are `undefined`.

This blocks the app's first user action: entering the lobby as a guest.

## Reproduction

1. Start Postgres with `pnpm db:up`.
2. Start the API with `pnpm --filter @cryptopoker/api dev`.
3. Start the web app with `NEXT_PUBLIC_API_BASE_URL=http://localhost:4000 pnpm --filter @cryptopoker/web dev`.
4. Open `http://localhost:3000`.
5. Enter a display name and click `Take a Seat`.

## Actual behavior

The welcome screen stays in place and shows:

`The API is not reachable yet. Start the API server and try again.`

The API is reachable, but `POST /players` returns `500`.

Observed server error:

```text
TypeError: Cannot read properties of undefined (reading 'createGuestSession')
    at PlayersController.createGuestSession (.../apps/api/src/sessions/players.controller.ts:25:35)
```

`GET /players/current` can also fail with:

```text
TypeError: Cannot read properties of undefined (reading 'findPlayerBySession')
    at CurrentPlayer.require (.../apps/api/src/sessions/current-player.ts:13:34)
```

## Expected behavior

The API dev command should behave like the compiled server: `POST /players` should create a guest session, set the session cookie, and return the current Player.

## Notes

The compiled API path works after `pnpm --filter @cryptopoker/api build` and `pnpm --filter @cryptopoker/api start`, which suggests the dev runner is not preserving the decorator metadata Nest needs for constructor injection.
