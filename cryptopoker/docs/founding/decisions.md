# Founding Decision Log

## ADR-001: Documentation-First Bootstrap

Date: 2026-04-13
Status: Accepted

Context:
The repository starts from an empty workspace and needs a clear baseline for product and engineering direction.

Decision:
Establish founding docs first (vision, principles, architecture, roadmap) before implementation begins.

Consequences:
- New contributors get aligned context quickly.
- Early implementation work should reference and refine these docs.
- Future architecture decisions should be recorded as incremental ADRs.
