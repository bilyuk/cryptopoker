# Sign Out Does Not Clear Server Session

> *This was generated by AI during triage.*

Status: ready-for-agent

Category: bug

## What is broken

Clicking `Sign out` only changes the local screen to the welcome view. The httpOnly session cookie remains valid, so reloading immediately signs the same Player back into the lobby.

## Reproduction

1. Run the compiled API server and the web app.
2. Open `http://localhost:3000`.
3. Enter as a guest with display name `codex_tester`.
4. Click `Sign out`.
5. Confirm the welcome screen appears.
6. Reload the page.

## Actual behavior

The app returns to the lobby automatically and shows:

`Start a hand, codex_tester.`

## Expected behavior

After sign-out, reload should keep the user on the welcome screen unless they intentionally create or resume a session.

## Acceptance criteria

- [ ] Add a server-backed sign-out endpoint or another explicit session-clearing mechanism.
- [ ] The frontend calls it from `Sign out`.
- [ ] The session cookie is expired or invalidated.
- [ ] Reloading after sign-out remains on the welcome screen.
- [ ] Tests cover sign-out followed by `GET /players/current`.
