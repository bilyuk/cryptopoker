# Trust model disclosure for Blockchain-Backed Rooms

Blockchain-backed Rooms must disclose their trust model plainly: funds are non-custodial against the platform, while checkout amounts are Host-arbitrated through the Room Host or Room Settlement Key. The product should never describe these Rooms as trustless; instead it should frame the model as familiar private poker host trust with on-chain escrow, signatures, and transaction history.

Disclosure should appear before a Player funds a Buy-In, persist subtly inside the Room, and appear during Room creation when the Host delegates a Room Settlement Key. Checkout receipts should show the payout amount, the authorizing Host or settlement delegate, and the on-chain transaction so the trust model is visible in operation as well as consent.
