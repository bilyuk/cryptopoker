# Waiting Room Renders Placeholder Seat Data

> *This was generated by AI during triage.*

Status: ready-for-agent

Category: bug

## What is broken

The waiting room mixes server-backed room settings with hard-coded placeholder seating copy.

After creating a new room, the server-backed room summary shows `Seats 0/6`, but the waiting-room side panel says `3 / 6`, lists `riverrat · host`, `magpie`, and `kings_up`, and says `3 seated`.

## Reproduction

1. Enter the app as a guest named `codex_tester`.
2. Create a private room named `Codex Test Room`.
3. Observe the waiting room.

## Actual behavior

The same screen shows conflicting seat state:

- Room settings: `Seats 0/6`
- Invite helper copy: `Anyone with this link can join. 3 of 6 seats filled.`
- At the table: `3 / 6`
- Seat list: `riverrat · host`, `magpie`, `kings_up`
- Footer copy: `Need at least 2 players. 3 seated. Ready when you are.`

## Expected behavior

Waiting-room occupancy, seat labels, stack values, and readiness copy should come from the server-backed room state.

## Acceptance criteria

- [ ] The waiting room renders `room.seats` instead of fixed placeholder players.
- [ ] Seat count is consistent across the room card, settings panel, table panel, invite copy, and footer copy.
- [ ] New rooms do not show unrelated sample players.
- [ ] Tests cover a newly created empty room and a room with occupied seats.
