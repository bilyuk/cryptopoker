# Founding Principles

1. Trust Is a Product Feature
Design decisions should increase user confidence in game integrity and fund safety.

2. Security Before Velocity
No launch milestone is worth bypassing custody or authentication controls.

3. Deterministic Core, Observable Edges
Game outcomes and fund movements must be reproducible and auditable.

4. Compliance Is Part of Architecture
Jurisdiction and policy constraints must shape product boundaries early.

5. Operational Simplicity
Prefer architectures the team can run safely with clear on-call ownership.

6. Explicit Ownership
Every service and critical workflow has a named technical owner.
